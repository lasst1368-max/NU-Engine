# Code of Conduct

## Short Version

Build well. Argue clearly. Respect people. Keep it technical.

## Expected Behavior

- be direct without being hostile
- critique code and ideas, not people
- keep discussion grounded in real engineering tradeoffs
- assume good faith until there is a concrete reason not to
- keep contributions attributable and honest

## Unacceptable Behavior

- harassment or intimidation
- personal attacks
- deliberate misrepresentation of authorship or contribution
- spam, trolling, or intentionally low-value disruption
- public disclosure of private security reports before a fix exists

## Project Culture

NU is an engineering-first project. The standard is clarity, rigor, and useful execution.

That means:

- vague claims should be backed by specifics
- architecture arguments should be coherent
- changes should move the engine forward, not just add noise

## Enforcement

Maintainers may edit, hide, lock, reject, or remove content and contributions that violate this
code of conduct.

Severe or repeated violations may result in removal from project spaces.
